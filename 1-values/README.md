# Values

The first letter of VAVPAH. Your ranked top values — the things that pressure-test every new project, every weekly check-in, every "should I take this on?" decision.

## What lives here

- **`values.md`** — your ranked list of top values, plus the menu of 30 candidates the system ships for the values exercise.

## How values get used

Values are the **strongest authority in the system.** When pushing back on a new project ("does this serve what you said matters?"), the agent cites values first. When you log an insight that contradicts a value, the system flags it during weekly check-in.

That means the order matters. Two values might both be true for you ("Health" and "Career") but one of them outranks the other when they conflict — and they will conflict, regularly. The exercise of *ranking* is what makes the values useful; an unranked list is much weaker as a decision aid.

## The values exercise

The shipped `values.md` includes a starter menu of 30 candidate values (love, faith, family, health, growth, learning, freedom, creativity, financial-stability, etc.). The exercise is:

1. Skim the menu. Cross off anything that doesn't resonate.
2. From what's left, pick your top 10–15.
3. Rank them.
4. Write a one-line "what this means in practice" for each of your top 5.

You don't have to do this in one sitting. Many people get to a draft, sit with it for a week, and revise. The system uses whatever's in `values.md` at the time — if you're still iterating, that's fine.

## When to revisit

- Quarterly check-ins surface values for a quick "still true?" pass.
- Major life changes (job change, move, new family member) often warrant a re-ranking.
- If you find yourself repeatedly pushing back on the system's pushback ("yes I know it doesn't serve Family but I want to do it anyway"), check whether Family is actually as high-ranked as `values.md` says.
